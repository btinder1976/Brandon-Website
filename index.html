Mission Control

The decision system.

Every project should move through the same simple launch sequence so it does not stay stuck in orbit.

  1. Capture the signal. Write the idea in one plain sentence.
  2. Pick the mission. Decide if it should make money, save time, teach, or create memories.
  3. Build version one. Create the smallest useful version.
  4. Test the build. Open it, use it, break it, and improve it.
  5. Launch or dock it. Ship the useful ones. Park the rest.
